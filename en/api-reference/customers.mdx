---
title: "Customers API — Create, List, Update, and Delete"
sidebarTitle: "Customers"
description: "Manage customers linked to your GoMoney account: create records with an automatic wallet, list them, update their data, and perform logical deletions via REST."
---

The Customers API lets you manage the external customers linked to your GoMoney account. Each customer has a tax document (CPF, CNPJ, or ID/Passport), an automatically generated wallet address, and a trackable balance. All operations require authentication via Bearer token and secret-key.

<Note>
  The resource hierarchy follows the order **customer → user → settings**: customers are always created under the context of the authenticated user and inherit the settings defined in their profile. The `userId` field present in each customer identifies the user who owns the record.
</Note>

## Base URL

```text
https://gmc-api.gomoney.me/api
```

## Authentication

All endpoints require the following headers:

<ParamField header="Authorization" type="string" required>
  Access token in the format `Bearer {{accessToken}}`.
</ParamField>

<ParamField header="secret-key" type="string" required>
  Account secret key, Base64-encoded: `{{secretKeyBase64}}`.
</ParamField>

## Available endpoints

| Method | Path | Purpose |
| --- | --- | --- |
| `POST` | `/customers` | Creates a new customer |
| `GET` | `/customers/all` | Lists all active customers |
| `GET` | `/customers/{customerId}` | Retrieves a specific customer by ID |
| `PATCH` | `/customers/{customerId}` | Updates a customer |
| `DELETE` | `/customers/{customerId}` | Logically removes a customer |

---

## POST /customers

Creates a new customer associated with the authenticated user. Once created, a wallet is automatically generated for the customer.

<ParamField header="Authorization" type="string" required>
  `Bearer {{accessToken}}`
</ParamField>

<ParamField header="secret-key" type="string" required>
  `{{secretKeyBase64}}`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  `application/json`
</ParamField>

### Body

<ParamField body="taxId" type="string" required>
  Customer's tax document (CPF, CNPJ, or ID/Passport).
</ParamField>

<ParamField body="accountType" type="string" required>
  Customer's account type. Possible values: `PF` (individual) or `PJ` (business).
</ParamField>

<ParamField body="name" type="string" required>
  Customer's name (legal/company name, in the case of `PJ`).
</ParamField>

<ParamField body="email" type="string" required>
  Customer's contact email.
</ParamField>

<ParamField body="phone" type="string" required>
  Customer's contact phone number (area code + number, no formatting).
</ParamField>

<ParamField body="birthDate" type="string">
  Customer's date of birth. Applicable only when `accountType` is `PF`. Format `YYYY-MM-DD`.
</ParamField>

    <ParamField body="phoneDdi" type="string">
      Company's phone country calling code (e.g. `+55`).

    <ParamField body="website" type="string">
      Company's website.

  </Expandable>
</ParamField>

<ParamField body="partners" type="object[]">

  <Expandable title="properties of each document">

## PATCH /customers/{customerId}

Partially updates the data of an existing customer. Only the fields sent in the body will be modified. You can also send the `company`, `partners`, `address`, `country`, and `documents` objects for updates.
