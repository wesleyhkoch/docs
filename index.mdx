Bem-vindo à documentação da GoMoney API. A GoMoney API é uma API de integração fintech brasileira que permite autenticar com segurança, gerenciar seus clientes e recarregar saldos de tokens GMC. Esta documentação cobre tudo, desde a configuração inicial e autenticação até o gerenciamento de clientes e a referência completa da API.

## Fluxo de Integração

Siga as seis etapas abaixo para sair do zero até a sua primeira chamada à API bem-sucedida.

<Steps>
  <Step title="Crie um Acesso de Integração">
    Acesse a plataforma GoMoney e crie um acesso do tipo **INTEGRATION** para o seu projeto com a permissão **INTEGRATION\_ONLY**. Esse é o único tipo de acesso autorizado a consumir os endpoints da API de integração.
  </Step>
  <Step title="Solicite as Credenciais">
    Entre em contato com o suporte GoMoney informando o identificador do acesso criado, o domínio da sua aplicação e a carteira operacional que será vinculada. O suporte abrirá o processo de provisionamento de credenciais.
  </Step>
  <Step title="Receba as Credenciais">
    Você receberá seu **clientKey** e **clientSecret** exclusivamente por e-mail do time de suporte GoMoney. Armazene essas credenciais com segurança e nunca as exponha em repositórios públicos ou logs.
  </Step>
  <Step title="Habilite a Integração">
    Com as credenciais em mãos, configure os headers de autenticação: encode `clientKey:clientSecret` em Base64 para o header `secret-key` e utilize o token JWT obtido via login no header `Authorization: Bearer <token>`.
  </Step>
  <Step title="Cadastre os Clientes">
    Antes de adicionar qualquer saldo, registre seus usuários finais como clientes na plataforma. O cadastro prévio é obrigatório — a API não permite adicionar saldo a clientes não registrados.
  </Step>
  <Step title="Adicione Saldos">
    Com os clientes cadastrados, realize operações de adição de saldo GMC. Cada operação deve estar vinculada a um cliente para garantir rastreabilidade e conciliação correta das transações.
  </Step>
</Steps>
