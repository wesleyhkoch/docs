This page describes the GoMoney API endpoints for **adding balance to customers**. Three operations are available: create a balance addition, list all additions for the authenticated user, and query a specific addition by ID.

**Base URL:** `https://gmc-api.gomoney.me/api`

## Authentication

All requests require the following headers:

| Header          | Value                                                                    |
| --------------- | ------------------------------------------------------------------------ |
| `Authorization` | `Bearer {{accessToken}}`                                                 |
| `secret-key`    | `{{secretKeyBase64}}`                                                    |
| `Origin`        | `{{allowedOrigin}}` - must match the domain approved by support          |

---

## Available Endpoints

| Method | Path                                            | Purpose                                  |
| ------ | ----------------------------------------------- | ---------------------------------------- |
| POST   | `/transactions/add-balance`                     | Adds balance to a customer               |
| GET    | `/transactions/add-balance/all`                 | Lists additions for the authenticated user |
| GET    | `/transactions/add-balance/{transactionId}`     | Queries a specific addition by ID        |

---

## POST /transactions/add-balance - Add balance

Creates a new balance addition transaction for a customer.

```http
POST https://gmc-api.gomoney.me/api/transactions/add-balance
Authorization: Bearer {{accessToken}}
secret-key: {{secretKeyBase64}}
Origin: {{allowedOrigin}}
Content-Type: application/json
```

### Request Body Parameters

<ParamField body="amount" type="number" required>
  Amount of the addition to be performed.
</ParamField>

<ParamField body="customerIdentifier" type="string" required>
  Identifier of the recipient customer. It can be the internal `_id` or the `taxId` (CPF/CNPJ).
</ParamField>

<ParamField body="currency" type="enum" required>
  Currency of the provided amount. Accepted values: `BRL`, `USD`, `GMC`, `USDT`.
</ParamField>

### Request Example

```json
{
  "items": [
    {
      "amount": 1000,
      "customerIdentifier": "66f0b1b5c1f7b7dc0ce30001",
      "currency": "GMC"
    }
  ]
}
```

### Operational Rules

<Note>
  - The `secret-key` must belong to the same user as the Bearer token.
  - The `Origin` header must exactly match the domain approved by support.
  - The `customerIdentifier` field is required and can be the customer's `_id` or `taxId`.
  - Fees follow the hierarchy: customer -> user -> global settings.
  - The operational wallet must be registered and approved before additions are made.
</Note>

### Response Codes

| Code | Description                                                                       |
| ------ | ------------------------------------------------------------------------------ |
| `201`  | Addition created successfully                                                    |
| `400`  | Invalid payload, inconsistent value, or missing/invalid `customerIdentifier`     |
| `401`  | Invalid token, `secret-key`, or `Origin`                                        |
| `403`  | User lacks permission to operate the integration                                |
| `503`  | GMC flow temporarily unavailable                                                |

## GET /transactions/add-balance/all - List additions

Returns all balance addition transactions linked to the authenticated user, with support for date-range filtering.

```http
GET https://gmc-api.gomoney.me/api/transactions/add-balance/all
Authorization: Bearer {{accessToken}}
secret-key: {{secretKeyBase64}}
```

### Query Parameters

<ParamField query="initialDate" type="string">
  Start date of the filter, in `YYYY-MM-DD` format. Example: `2026-07-01`.
</ParamField>

<ParamField query="finalDate" type="string">
  End date of the filter, in `YYYY-MM-DD` format. Example: `2026-07-31`.
</ParamField>

### Successful Response - 200

Returns an array of `balanceTransaction` objects, each with the same structure described in the POST endpoint above.

### Response Codes

| Code | Description                               |
| ------ | -------------------------------------- |
| `200`  | List returned successfully              |
| `401`  | Invalid token or `secret-key`           |

## GET /transactions/add-balance/\{transactionId\} - Query by ID

Returns the details of a specific balance addition transaction, identified by its internal ID.

### Path Parameter

<ParamField path="transactionId" type="string" required>
  Internal identifier of the balance addition (`_id` returned on creation).
</ParamField>

```http
GET https://gmc-api.gomoney.me/api/transactions/add-balance/{{transactionId}}
Authorization: Bearer {{accessToken}}
secret-key: {{secretKeyBase64}}
```

### Successful Response - 200

Returns the corresponding `balanceTransaction` object, with the same structure described in the POST endpoint above.

### Response Codes

| Code | Description                                                             |
| ------ | -------------------------------------------------------------------- |
| `200`  | Addition found and returned successfully                               |
| `401`  | Invalid token or `secret-key`                                          |
| `417`  | Addition not found or not linked to the authenticated user             |
