Esta página descreve os endpoints da GoMoney API destinados à **adição de saldo em customers**. São três operações disponíveis: criar uma adição de saldo, listar todas as adições do usuário autenticado e consultar uma adição específica por ID.

**URL base:** `https://gmc-api.gomoney.me/api`

## Autenticação

Todas as requisições exigem os seguintes cabeçalhos:

| Cabeçalho       | Valor                                                                 |
| --------------- | --------------------------------------------------------------------- |
| `Authorization` | `Bearer {{accessToken}}`                                              |
| `secret-key`    | `{{secretKeyBase64}}`                                                 |
| `Origin`        | `{{allowedOrigin}}` — deve corresponder ao domínio aprovado pelo suporte |

---

## Endpoints disponíveis

| Método | Caminho                                    | Finalidade                               |
| ------ | ------------------------------------------ | ---------------------------------------- |
| POST   | `/transactions/add-balance`                | Adiciona saldo em um customer            |
| GET    | `/transactions/add-balance/all`            | Lista adições do usuário autenticado     |
| GET    | `/transactions/add-balance/{id}`           | Consulta adição específica por ID        |

---

## POST /transactions/add-balance — Adicionar saldo

Cria uma nova transação de adição de saldo para um customer.

```http
POST https://gmc-api.gomoney.me/api/transactions/add-balance
Authorization: Bearer {{accessToken}}
secret-key: {{secretKeyBase64}}
Origin: {{allowedOrigin}}
Content-Type: application/json
```

### Parâmetros do corpo da requisição

<ParamField body="amount" type="number" required>
  Valor da adição a ser realizada.
</ParamField>

<ParamField body="customerIdentifier" type="string" required>
  Identificador do customer destinatário. Pode ser o `_id` interno ou o `taxId` (CPF/CNPJ).
</ParamField>

<ParamField body="currency" type="enum" required>
  Moeda do valor informado. Valores aceitos: `BRL`, `USD`, `GMC`, `USDT`.
</ParamField>

### Exemplo de requisição

```json
{
  "amount": 1000,
  "customerIdentifier": "66f0b1b5c1f7b7dc0ce30001",
  "currency": "GMC"
}
```

### Regras operacionais

<Note>
  - A `secret-key` deve pertencer ao mesmo usuário do Bearer token.
  - O cabeçalho `Origin` deve corresponder exatamente ao domínio aprovado pelo suporte.
  - O campo `customerIdentifier` é obrigatório e pode ser o `_id` ou o `taxId` do customer.
  - As taxas seguem a hierarquia: customer → usuário → configurações globais.
  - A carteira operacional deve estar cadastrada e aprovada antes de realizar adições.
</Note>

### Códigos de resposta

| Código | Descrição                                                                      |
| ------ | ------------------------------------------------------------------------------ |
| `201`  | Adição criada com sucesso                                                      |
| `400`  | Payload inválido, valor inconsistente ou `customerIdentifier` ausente/inválido |
| `401`  | Token, `secret-key` ou `Origin` inválidos                                     |
| `403`  | Usuário sem permissão para operar a integração                                 |
| `503`  | Fluxo GMC indisponível temporariamente                                         |

## GET /transactions/add-balance/all — Listar adições

Retorna todas as transações de adição de saldo vinculadas ao usuário autenticado, com suporte a filtro por período.

```http
GET https://gmc-api.gomoney.me/api/transactions/add-balance/all
Authorization: Bearer {{accessToken}}
secret-key: {{secretKeyBase64}}
```

### Parâmetros de consulta (query)

<ParamField query="initialDate" type="string">
  Data inicial do filtro, no formato `YYYY-MM-DD`. Exemplo: `2026-07-01`.
</ParamField>

<ParamField query="finalDate" type="string">
  Data final do filtro, no formato `YYYY-MM-DD`. Exemplo: `2026-07-31`.
</ParamField>

### Resposta de sucesso — 200

Retorna um array de objetos `balanceTransaction`, cada um com a mesma estrutura descrita no endpoint POST acima.

### Códigos de resposta

| Código | Descrição                              |
| ------ | -------------------------------------- |
| `200`  | Lista retornada com sucesso            |
| `401`  | Token ou `secret-key` inválidos        |

## GET /transactions/add-balance/\{transactionId\} — Consultar por ID

Retorna os detalhes de uma transação de adição de saldo específica, identificada pelo seu ID interno.

### Parâmetro de caminho (path)

<ParamField path="transactionId" type="string" required>
  Identificador interno da adição de saldo (`_id` retornado na criação).
</ParamField>

```http
GET https://gmc-api.gomoney.me/api/transactions/add-balance/{{transactionId}}
Authorization: Bearer {{accessToken}}
secret-key: {{secretKeyBase64}}
```

### Resposta de sucesso — 200

Retorna o objeto `balanceTransaction` correspondente, com a mesma estrutura descrita no endpoint POST acima.

### Códigos de resposta

| Código | Descrição                                                            |
| ------ | -------------------------------------------------------------------- |
| `200`  | Adição encontrada e retornada com sucesso                            |
| `401`  | Token ou `secret-key` inválidos                                      |
| `417`  | Adição não encontrada ou sem vínculo com o usuário autenticado       |
