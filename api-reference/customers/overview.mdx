A API de Customers permite que você gerencie os clientes externos vinculados à sua conta GoMoney. Cada customer possui um documento fiscal (CPF ou CNPJ ou ID/Passaporte), um endereço de wallet gerado automaticamente e um saldo rastreável. Todas as operações exigem autenticação via token Bearer e secret-key.

<Note>
  A hierarquia de recursos segue a ordem **customer → user → settings**: customers são sempre criados sob o contexto do usuário autenticado e herdam as configurações definidas em seu perfil. O campo `userId` presente em cada customer identifica o usuário proprietário do registro.
</Note>

## Base URL

```text
https://gmc-api.gomoney.me/api
```

## Autenticação

Todos os endpoints exigem os seguintes cabeçalhos:

| Cabeçalho       | Valor                     |
| --------------- | -------------------------- |
| `Authorization` | `Bearer {{accessToken}}`   |
| `secret-key`    | `{{secretKeyBase64}}`      |

<Tip>
  Não sabe como obter `accessToken` e `secretKeyBase64`? Veja [Autenticação](/api-reference/authentication) — o próprio playground de login te devolve o token que você pode colar nos campos de autenticação abaixo.
</Tip>
